 # Pharmacy Management System - SQL Project

# Project Overview

The Pharmacy Management System project was developed to simulate real-world pharmacy operations and demonstrate how SQL can be used to manage and analyze business data.

The project focuses on sales analysis, inventory management, customer behavior analysis, medicine performance tracking, and supplier monitoring. Using SQL queries, meaningful insights were generated to help improve business efficiency and support data-driven decisions.

# Business Problem
Pharmacies generate large amounts of data related to medicine sales, inventory levels, customer purchases, and supplier management. However, without proper analysis, it becomes difficult to monitor business performance, identify top-selling medicines, track inventory status, and understand customer purchasing behavior.
The objective of this project is to design a Pharmacy Management System database and perform SQL-based analysis to generate actionable business insights that support operational and strategic decision-making.

# Project Objectives
- Design a relational database for pharmacy operations.
- Store and manage information related to medicines, customers, sales, suppliers, and inventory.
- Analyze medicine sales performance and revenue generation.
- Identify top-selling medicines and customer purchasing patterns.
- Monitor inventory levels and stock availability.
- Generate business reports to support operational decision-making.
- Apply SQL concepts such as joins, aggregations, subqueries, grouping, filtering, and business reporting.

## Sample Analytical Queries

### Top Selling Medicines
```sql
SELECT MedicineName,
       SUM(Quantity) AS TotalSold
FROM Sales
GROUP BY MedicineName
ORDER BY TotalSold DESC;
```
## Revenue Analysis

```sql
SELECT SUM(TotalAmount) AS TotalRevenue
FROM Sales;
```
### Customer Purchase Analysis

```sql
SELECT CustomerID,
       COUNT(*) AS TotalPurchases
FROM Sales
GROUP BY CustomerID
ORDER BY TotalPurchases DESC;
```
